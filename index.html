Problem & goal

Clinicians and patients often lack a fast, data-backed first read on symptoms. The goal was a decision-support tool that could suggest likely conditions from reported symptoms, accurately enough to be genuinely useful as a first pass.

Solution

A neural network trained on a 41-disease, 117-symptom Kaggle dataset, combined with knowledge-based reasoning and optimization logic, exposed through a REST API for downstream use.

Role & approach

  • End-to-end build: preprocessing, feature engineering, hyperparameter tuning
  • Designed the REST API and full-stack architecture
  • Applied formal software engineering methodology for scalability

A working full-stack platform reaching 99.46% test accuracy — evidence that ML rigor and real software engineering discipline can come from the same build.
